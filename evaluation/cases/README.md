# cases

이 폴더는 regression set 운영용입니다. 목표는 평가 때마다 다른 문서를 임의로 고르는 것이 아니라, 문서 타입별 대표 케이스를 고정해두는 것입니다.

git tracked 파일에는 실제 제목을 쓰지 않습니다. 케이스 파일명과 본문은 모두 익명 ID 기반으로 유지합니다. 실제 매핑은 로컬 전용 `evaluation/private/case-map.local.md`에만 둡니다.

공개 레포는 방법론과 케이스 구조만 공유합니다. 실제 benchmark 문서와 실제 run 결과는 각 사용자 환경에서 로컬 전용으로 채우는 방식이 기본입니다.

## 권장 구성

아래 수치는 목표 상태 기준입니다. 공개 tracked 케이스는 핵심 문서 타입부터 먼저 채우고, 나머지는 필요에 따라 점진적으로 추가합니다.

- 전략 메모: 2개
- PRD / 서비스기획: 3개
- 상위기획 / 컨셉 기획: 1개
- 유즈케이스 / 버티컬 워크플로: 2개
- 허브 / 인덱스 페이지: 1개
- 주간 업데이트 / 원페이저: 1개

## 각 케이스에 같이 남길 정보

- 익명 케이스 ID
- 문서 타입
- 일반화된 설명
- 독자
- 어떤 모드로 자주 평가하는지
- 과거 반복 실패 코드
- 공개 tracked 케이스인지, 로컬 전용 확장 케이스인지
- 실제 문서명은 local-only mapping에만 기록

## 운영 원칙

- 같은 문서를 반복 사용해야 변화 비교 가능
- 너무 잘 된 문서만 모으지 말 것
- 반복 실패가 있었던 문서를 오히려 남길 것
- 스킬 수정 후에는 최소 2개 이상의 기존 케이스로 회귀 확인
