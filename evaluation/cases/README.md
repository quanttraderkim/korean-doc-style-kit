# cases

이 폴더는 regression set 운영 방식을 설명하는 공개 영역입니다. public git에는 실제 케이스 카드까지 두지 않고, 구조 설명과 템플릿만 남깁니다.

실제 케이스 카드는 각 사용자 환경에서 로컬 전용 `evaluation/cases/*.local.md`로 운영하는 것을 기본으로 합니다. 실제 문서명, URL, page id, 내부 프로젝트명은 git tracked 파일에 넣지 않습니다.

## public repo에 남기는 것

- 케이스 운영 원칙
- 권장 문서 타입 분포
- 케이스 카드 템플릿

즉 public repo는 `어떻게 케이스를 잡고 회귀를 돌릴 것인가`만 공유합니다. 실제 benchmark 문서 세트와 개별 케이스 카드는 로컬에서만 유지합니다.

## 로컬에서 관리하는 것

- `STRAT-001.local.md` 같은 실제 케이스 카드
- 실제 문서명, URL, page id 매핑
- 점수판과 날짜별 run 결과

실제 매핑은 로컬 전용 `evaluation/private/case-map.local.md`에만 둡니다.

## 권장 범위

회귀 세트는 핵심 문서 타입을 빠짐없이 포함하는 것이 중요합니다. 정확한 개수보다 `문서 타입 다양성`과 `반복 비교 가능성`이 더 중요합니다.

보통 아래 문서군을 먼저 확보하면 충분합니다.

- 전략 메모
- PRD / 서비스기획
- 상위기획 / 컨셉 기획
- 유즈케이스 / 버티컬 워크플로
- 허브 / 인덱스 페이지
- 주간 업데이트 / 원페이저

## 케이스 카드에 남길 정보

- 익명 케이스 ID
- 문서 타입
- 일반화된 설명
- 독자
- 기본 적용 모드
- 과거 반복 실패 코드

구조 예시는 [`case.template.md`](./case.template.md)를 따릅니다.

## 운영 원칙

- 같은 문서를 반복 사용해야 변화 비교 가능
- 너무 잘 된 문서만 모으지 말 것
- 반복 실패가 있었던 문서를 오히려 남길 것
- 스킬 수정 후에는 최소 2개 이상의 기존 케이스로 회귀 확인
