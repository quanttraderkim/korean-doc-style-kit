# results

이 폴더는 평가 로그 운영 방식을 설명하는 곳입니다. 실제 평가 로그는 git tracked 영역에 두지 않고 로컬 전용 파일로만 관리합니다. 목적은 “이번에 느낌이 좋았음”이 아니라, 어떤 수정이 어떤 문서 타입에서 실제로 개선을 만들었는지 로컬에서 누적하는 것입니다.

## 파일 권장 형식

- 파일명: `YYYY-MM-DD.local.md`
- 한 파일 안에 여러 문서 평가를 묶어도 무방

## 각 로그에 남길 최소 정보

- 케이스 ID
- 문서 타입
- 적용 모드
- 실행 라운드
- 비교 기준
- 직전 대비 규칙 변경점
- rubric 점수표
- 실패 코드
- 다음 수정 방향

실제 제목, URL, page id는 넣지 않습니다. 실제 매핑은 로컬 전용 파일에만 둡니다. 실제 평가 로그도 `evaluation/results/*.local.md`에만 둡니다.

## 좋은 로그의 기준

- 길지 않아도 됨
- 같은 실패 코드가 반복되는지 보이면 충분
- 점수 변화와 실패 코드가 같이 있어야 함
- rerun이면 `무엇을 바꿔서` 좋아졌는지 짧게 남겨야 함
