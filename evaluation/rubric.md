# rubric

이 점수표는 감상평을 대신하는 것이 아니라, `Skill Off / Skill On` 비교를 같은 축에서 남기기 위한 최소 기준입니다.

## 점수 방식

- `0`: 문제 큼 / 문서 목적에 맞지 않음
- `1`: 부분 충족 / 읽히지만 아쉬움 큼
- `2`: 충분히 충족 / 실무 사용 가능

## 평가 축

| 항목 | 질문 | 점수 |
| --- | --- | --- |
| 첫 화면 | 목적, 핵심 판단, 읽어야 하는 이유가 첫 화면에서 보이는가 | 0~2 |
| 문체 | 공손체, 기계적 `~함`, 긴 `~한다` 설명문 없이 메모체가 유지되는가 | 0~2 |
| 구조 | 제목이 메시지 역할을 하고, 문단/섹션 구성이 스캔 가능하게 잡혀 있는가 | 0~2 |
| 표 / 불릿 | 표와 불릿이 필요한 곳에만 쓰이고, 가짜 개괄식이 없는가 | 0~2 |
| 문서 타입 적합성 | 전략 메모, PRD, 유즈케이스, 허브 문서 등 해당 타입의 기대 구조에 맞는가 | 0~2 |

총점은 `10점 만점` 기준으로 보되, 총점보다 어떤 항목이 반복적으로 낮은지가 더 중요합니다.

## 해석 기준

- `+2 이상`: 스킬 효과 분명
- `0 ~ +1`: 일부 개선, 규칙 보강 여지 큼
- `음수`: 스킬 또는 프리셋 적용 실패 가능성 높음

점수만 기록하고 끝내지 말고, 낮은 항목은 반드시 [`failure-taxonomy.md`](./failure-taxonomy.md) 코드와 연결하는 것을 권장합니다.
